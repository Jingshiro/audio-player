# 已部署用户更新指南

> 本次更新包含 10 项修复（5 项安全性 + 5 项易用性），其中有 **2 个破坏性变更** 需要手动操作。

---

## ⚠️ 破坏性变更（必须操作）

### 1. 数据库路径迁移

本次更新将数据库默认路径从 `./data/music.db` 改为 `./db/music.db`，避免数据库文件被静态目录暴露下载。

**操作步骤**：

```bash
# 1. 停止服务
docker compose down

# 2. 创建新目录并复制数据库
mkdir -p db
cp data/music.db db/music.db

# 3. 更新代码（git pull 或重新构建镜像）
git pull

# 4. 重启服务
docker compose up -d

# 5. 确认服务正常后，删除旧数据库
rm data/music.db
```

> 如果你使用了自定义 `DATABASE_URL` 环境变量，则无需迁移，路径仍以你的配置为准。

### 2. 密码哈希算法升级（bcrypt）

本次更新将密码哈希从 SHA-256 升级为 bcrypt。旧的 SHA-256 哈希无法通过 bcrypt 验证，**所有用户需要重新登录**。

**操作步骤**：

```bash
# 方案 A：删除密码记录，让系统重新初始化默认密码
# 1. 停止服务
docker compose down

# 2. 删除数据库中的密码记录（使用 sqlite3）
sqlite3 db/music.db "DELETE FROM settings WHERE key = 'auth_password';"

# 3. 重启服务
docker compose up -d

# 4. 使用默认密码 admin 登录，然后立即修改密码
```

```bash
# 方案 B：直接删除整个数据库（会丢失所有音频/台词数据，仅适合测试环境）
rm db/music.db
docker compose up -d
```

> 如果你设置了 `AUTH_PASSWORD` 环境变量，重启后系统会用该值初始化默认密码。

## ✅ 无需操作的变更

以下变更对用户透明，更新代码后自动生效：

| 变更 | 说明 |
|------|------|
| API 路由认证保护 | 所有 API 接口现在都需要登录 token，未登录返回 401。已登录用户无感知 |
| STT 日志脱敏 | 后端日志不再输出完整 URL（可能含 API Key），只记录域名 |
| Nginx 目录列表关闭 | `/data/` 路径不再显示文件列表 |
| 音频搜索修复 | 搜索框现在能正常过滤音频列表 |
| 循环/随机播放 | 播放器新增循环模式切换按钮（不循环 → 列表循环 → 单曲循环 → 随机） |
| 空状态引导 | 音频库/台词库为空时显示导入按钮 |
| LRC 格式提示 | 编辑台词时 textarea 显示 LRC 格式示例 |
| PlayerView 优化 | 合并重复的初始化逻辑，移除冗余数据加载 |

```bash

# 3. 迁移数据库
mkdir -p db

```

## 常见问题

**Q: 更新后登录提示"密码错误"怎么办？**
A: 因为密码哈希算法升级，旧密码无法验证。请按上述"密码哈希算法升级"步骤重置密码。

**Q: 更新后音频库为空怎么办？**
A: 检查数据库文件是否已从 `data/music.db` 复制到 `db/music.db`。

**Q: 我用了自定义 DATABASE_URL，需要改吗？**
A: 不需要。自定义路径优先级高于默认值，你的配置不受影响。

**Q: 循环模式怎么用？**
A: 在播放器页面，点击"下一首"按钮左边的循环图标，依次切换：不循环 → 列表循环 → 单曲循环 → 随机播放。当前模式会自动保存。
